# Feature Specification: 规范相似性检查与冲突检测

**Feature Branch**: `001-similarity-conflict-detection`
**Created**: 2026-03-22
**Status**: Draft
**Input**: User description: "为 Steering Hub 平台实现规范相似性检查和冲突检测功能"

## Clarifications

### Session 2026-03-22

- Q: 前端如何感知异步检测任务完成？ → A: SSE（Server-Sent Events）— 服务端任务完成时推送事件，前端监听更新
- Q: 冲突检测的候选对范围如何界定？ → A: 同分类（categoryId 相同）或 tags 重叠率 ≥ 50% 的规范对——而非向量相似度高的规范对；相似检测（高 cosine → 冗余 → 建议合并）与冲突检测（同类 → 矛盾 → 建议修订）是两个独立维度

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 运行相似性检测并查看结果 (Priority: P1)

管理员进入"规范健康度"页面，点击"运行检测"按钮，系统对所有 active 状态的规范进行两两相似度分析。检测完成后，页面展示相似规范对列表，按相似度从高到低排序，每对规范旁标注相似原因（如标题相似、tags 重叠、内容语义相近）。

**Why this priority**: 相似性检测是核心功能，解决规范散落和重复问题，为合并操作提供依据，独立完整交付价值。

**Independent Test**: 在系统中存在若干 active 规范的情况下，点击"运行检测"，等待完成后可看到相似规范对列表，交付价值：发现潜在的重复规范。

**Acceptance Scenarios**:

1. **Given** 系统中存在至少 2 条 active 规范，**When** 管理员点击"运行检测"，**Then** 系统启动后台分析任务，页面显示检测进行中状态。
2. **Given** 检测任务完成，**When** 管理员查看结果页，**Then** 页面展示相似规范对列表，每对显示：两条规范标题、综合相似度分数（0–1）、相似原因标签（标题相似 / tags 重叠 / 关键词重叠 / 内容语义相近）。
3. **Given** 相似规范对列表已展示，**When** 管理员查看，**Then** 列表按相似度降序排列，相似度低于默认阈值（0.7）的规范对不显示。
4. **Given** 系统中所有 active 规范彼此相似度均低于阈值，**When** 检测完成，**Then** 列表为空，页面显示"未发现相似规范对"提示。
5. **Given** 相似规范对列表已展示，**When** 管理员在搜索栏输入标题关键词或选择分类，**Then** 列表即时过滤，仅显示规范A或规范B标题包含关键词、且规范A所属分类匹配的规范对；分类下拉选项从系统分类列表动态加载。

---

### User Story 2 - 查看规范内容对比 (Priority: P2)

管理员在相似规范对列表或冲突规范对列表中，点击某一规范对，进入左右分屏对比视图，并排展示两条规范的完整内容，方便判断是否需要合并或处理冲突。

**Why this priority**: 内容对比是合并和冲突处理操作的前置步骤，管理员需要看到全文才能做出准确判断。

**Independent Test**: 点击任意相似规范对，可在左右分屏中看到两条规范的完整内容，无需额外操作。

**Acceptance Scenarios**:

1. **Given** 相似规范对列表已展示，**When** 管理员点击某一规范对的"对比"按钮，**Then** 页面切换到左右分屏视图，左右两侧分别展示两条规范的标题、tags 和完整正文内容。
2. **Given** 处于分屏对比视图，**When** 管理员滚动一侧内容，**Then** 两侧内容独立滚动，互不干扰。

---

### User Story 3 - 运行冲突检测并查看冲突列表 (Priority: P2)

系统自动（或管理员手动触发）对同分类（categoryId 相同）或 tags 重叠率 ≥ 50% 的规范对进行 AI 语义冲突分析，识别内容相互矛盾的规范对，并展示冲突规范对列表，每对显示冲突点摘要。冲突检测与相似性检测相互独立，可分别单独触发。

**Why this priority**: 冲突检测解决比相似更严重的"指导意见矛盾"问题，防止 AI Agent 收到相互矛盾的指令，与相似性检测同等重要。

**Independent Test**: 在存在同分类或 tags 高度重叠的规范对的情况下，完成冲突检测后可看到冲突规范对列表（含冲突摘要），交付价值：识别会导致 AI Agent 行为不一致的矛盾规范。

**Acceptance Scenarios**:

1. **Given** 系统中存在至少 2 条 active 规范，**When** 冲突检测运行，**Then** 系统仅对同分类（categoryId 相同）或 tags 重叠率 ≥ 50% 的规范对进行 AI 语义分析，不对全量规范两两分析；冲突检测不依赖相似性检测是否已运行。
2. **Given** 冲突检测完成，**When** 管理员查看冲突规范对列表，**Then** 每对显示：两条规范标题、冲突点摘要（简述矛盾之处）、严重程度标记（高 / 中 / 低）。
3. **Given** 冲突检测完成，**When** 列表为空，**Then** 页面显示"未发现冲突规范"提示。

---

### User Story 4 - 合并相似规范 (Priority: P3)

管理员在分屏对比视图中，确认两条规范内容高度重合后，点击"合并"按钮，系统将两条规范的内容合并为一条新规范草稿，供管理员编辑确认后提交审批。

**Why this priority**: 合并是解决相似问题的最终手段，但依赖内容对比（P2）且需要审批流程，优先级低于检测和查看。

**Independent Test**: 对一对相似规范执行合并操作，可产生一条新的 draft 规范，并将原两条规范标记为 deprecated，验证规范总数变化。

**Acceptance Scenarios**:

1. **Given** 管理员处于分屏对比视图，**When** 点击"合并"按钮，**Then** 系统生成一条新的规范草稿，内容由 AI 整合两条原规范，初始状态为 draft。
2. **Given** 合并草稿已生成，**When** 管理员确认提交，**Then** 原两条规范状态变更为 deprecated，新合并规范进入审批流程（pending_review）。
3. **Given** 合并草稿已生成，**When** 管理员取消，**Then** 原两条规范状态不变，草稿丢弃。

---

### User Story 5 - 标记冲突处理状态 (Priority: P3)

管理员在冲突规范对列表中，对每对冲突规范标记处理状态：已知冲突（暂时接受此冲突，待后续处理）或非冲突（误报，忽略此对）。

**Why this priority**: 状态标记帮助管理员追踪冲突处理进度，避免同一冲突重复审查，是完整工作流的收尾步骤。

**Independent Test**: 对一对冲突规范标记为"非冲突（误报）"，刷新后该对不再显示在未处理列表中。

**Acceptance Scenarios**:

1. **Given** 冲突规范对列表展示，**When** 管理员点击某对规范旁的"标记处理"，**Then** 弹出操作选项：已知冲突 / 非冲突（误报）。
2. **Given** 管理员选择"非冲突（误报）"，**When** 确认，**Then** 该规范对从冲突列表隐藏，标记持久化保存，下次检测时不再对此对做冲突分析。
3. **Given** 管理员选择"已知冲突"，**When** 确认，**Then** 该规范对标记为已知冲突状态，在列表中显示已处理标记，不影响规范本身的 active 状态。

---

### Edge Cases

- 当规范数量为 0 或 1 条时，运行检测后直接显示"规范数量不足，无需检测"。
- 当检测任务正在运行时，再次点击"运行检测"，按钮禁用并提示"检测进行中"。
- 当 AI 冲突分析调用失败（网络超时、服务不可用），对应规范对标记为"分析失败"，不影响其他规范对的分析结果。
- 当两条相似规范中有一条在检测期间被删除或变更为非 active 状态，该规范对在结果中标记为"规范已变更，建议重新检测"。
- 当 active 规范数量较大（如 200+ 条）时，两两组合数量庞大，系统应有合理的进度反馈机制。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供"规范健康度"页面，管理员可在此页面触发检测并查看历史检测结果。
- **FR-002**: 系统 MUST 对所有 active 状态的规范执行两两相似度分析，综合以下四个维度计算综合相似度分数：标题文本相似度、tags 重叠率、关键词重叠率、内容向量余弦相似度。
- **FR-003**: 系统 MUST 仅展示综合相似度分数不低于配置阈值（默认 0.7）的规范对，并按分数降序排列。
- **FR-004**: 系统 MUST 为每对相似规范显示触发相似的具体维度标签（标题相似 / tags 重叠 / 关键词重叠 / 内容语义相近），说明相似原因。
- **FR-005**: 系统 MUST 仅对同分类（categoryId 相同）或 tags 重叠率 ≥ 50% 的规范对触发 AI 冲突分析，输出冲突规范对列表，每对含冲突点摘要和严重程度（高 / 中 / 低）；冲突检测与相似性检测相互独立，可单独触发。
- **FR-006**: 系统 MUST 支持管理员为每对冲突规范标记处理状态：已知冲突 或 非冲突（误报）。
- **FR-007**: 系统 MUST 支持对任意规范对（相似或冲突）进行左右分屏内容对比，展示两条规范的标题、tags 和完整正文。
- **FR-008**: 系统 MUST 支持管理员对相似规范对发起合并操作，由 AI 生成合并草稿，初始状态为 draft，须经现有审批流程后方可激活。
- **FR-009**: 合并操作完成提交后，系统 MUST 将原两条规范状态自动变更为 deprecated，新合并规范进入 pending_review 状态。
- **FR-010**: 检测任务 MUST 以异步方式执行，管理员可在检测运行期间查看页面其他内容，任务完成后通过 SSE（Server-Sent Events）向前端推送完成事件，前端监听后自动刷新结果；后端使用 Spring Boot `SseEmitter` 实现。
- **FR-011**: 系统 MUST 持久化最近一次检测结果，管理员刷新页面后仍可查看，结果有效期 24 小时。
- **FR-012**: 系统 MUST 显示每次检测的执行时间和完成时间，供管理员判断结果时效性。
- **FR-013**: 冲突标记状态 MUST 持久化保存，被标记为"非冲突（误报）"的规范对在后续检测中不再重复报告。

### Key Entities

- **健康度检测任务 (HealthCheckTask)**: 代表一次完整的检测执行，记录触发时间、完成时间、执行状态（进行中 / 已完成 / 失败）、检测到的相似对数量和冲突对数量。
- **相似规范对 (SimilarSpecPair)**: 两条规范的相似度分析结果，包含两条规范的引用、综合相似度分数（0–1）、各维度相似度分项、相似原因标签，关联至某次检测任务。
- **冲突规范对 (ConflictSpecPair)**: 两条规范的冲突分析结果，包含两条规范的引用、冲突点摘要文本、严重程度（高 / 中 / 低）、处理状态（未处理 / 已知冲突 / 非冲突误报）、处理时间，关联至某次检测任务。
- **合并草稿 (MergeDraft)**: 由合并操作生成的新规范草稿，包含 AI 生成的合并内容、源规范 A 和源规范 B 的引用，通过现有规范审批流程处理。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 对于包含 100 条 active 规范的系统，完整检测任务（相似性 + 冲突检测）在 3 分钟内完成。
- **SC-002**: 相似度计算覆盖所有 active 规范的全量两两组合，无遗漏。
- **SC-003**: 冲突检测的误报率（被管理员标记为"非冲突"的比例）不超过 30%（以真实使用 2 周的数据为基准）。
- **SC-004**: 合并操作端到端流程（发起合并 → 生成草稿 → 提交审批）可在 2 分钟内完成。
- **SC-005**: 检测结果页面在检测完成后自动更新，管理员无需手动刷新即可看到结果。
- **SC-006**: 该功能不影响现有规范管理、搜索、审批功能的响应时间（不超过 10% 的性能劣化）。

## Assumptions

- 相似度综合阈值默认为 0.7，不提供界面配置入口（可在后端配置文件调整），以降低首版复杂度。
- 冲突检测不对所有规范两两进行 AI 分析（O(n²) 成本过高），仅针对同分类（categoryId 相同）或 tags 重叠率 ≥ 50% 的规范对，以控制 AI 调用成本。相似性检测（高向量余弦 → 内容冗余 → 建议合并）与冲突检测（同类 → 内容矛盾 → 建议修订）是两个独立维度，候选池不同，可分别独立触发。
- 合并草稿由 AI 自动生成初稿，管理员可在提交前编辑；合并草稿使用现有规范编辑界面。
- 被标记为"非冲突（误报）"的规范对在后续检测中跳过 AI 冲突分析，但相似度分析仍会运行（相似度本身是客观计算结果）。
- "规范健康度"作为前端导航中的新菜单项，不改变现有菜单结构，仅新增入口。
- 检测结果有效期 24 小时后在界面提示"结果已过期，建议重新检测"，但不自动删除历史记录。
